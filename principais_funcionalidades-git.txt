git clone *url_do_repositório*: clona o repositório da url digitada para a pasta atual

git init: inicia um repositório git na pasta atual
git remote add *alias_do_repositório* *url_do_repositório*: conecta o repositório local com o repositório remoto
git status: mostra os arquivos que tiveram alterações que ainda não foram adicionadas ou commitadas
git checkout: lista as branchs
git checkout -b *nome_da_branch*: cria uma branch com o nome digitado e muda para ela
git add: monitora as alterações feitas no arquivo
git commit: salva as alterações feitas no arquivo
git checkout *nome_da_branch*: muda para a branch com o nome digitado
git fetch *nome_da_branch_remota*: sincroniza commits da branch remota
git rebase *branch_origem* *branch_destino*: junta commits da branch origem digitada aos da branch destino
git merge *nome_da_branch*: funde commits da branch com o nome passado para a branch atual
git push *alias_do_repositório* *nome_da_branch*: sincroniza o repositório local com o repositório remoto
git pull *alias_do_repositório* *nome_da_branch*: sincroniza o repositório remoto com o repositório local

git checkout -- *nome_do_arquivo*: desfaz modificação não monitorada
git reset HEAD *nome_do_arquivo*: desfaz modificação de status do arquivo com o nome digitado
git reset *hash_do_commit*: desfaz commit com o hash digitado
git revert *hash_do_commit*: reverte as mudanças do commit com o hash digitado

git stash: salva mudanças ainda não commitadas em uma branch temporária
git stash list: mostra as branchs temporárias
git stash pop: recupera último item das branchs temporárias
git stash apply *número_do_stash*: recupera a branch temporária com o número do stash digitado
git stash drop: deleta a branch temporária

git bisect start: inicia bisect, um processo de busca por alterações desejadas feitas em commits antigos
git bisect bad *hash_do_commit*: diz ao bisect que esse commit é ruim, logo, não é desejado
git bisec good *hash_do_commit*: diz ao bisect que esse commit é bom, logo, é desejado
git cherry-pick *hash_do_commit*: importa commit com o hash digitado

git ls-files: mostra os arquivos monitorados
git log: mostra o histórico de commits
git blame *nome_do_arquivo*: mostra informações sobre os commits (hash do commit, quem fez, quando foi feito e o que foi feito)

git tag *nome_da_versão*: faz um deploy do repositório com o nome digitado
git tag: mostra os deploys feitos
git diff *nome_da_versão*: mostra as diferenças entre a versão atual e a versão digitada
git whatchanged: mostra o git log e o que foi modificado em cada commit

vim ~/.gitconfig: abre arquivo de configuração do git
